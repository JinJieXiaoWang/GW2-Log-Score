# 运维手册

## 1. 系统监控

### 1.1 监控服务

系统提供了实时监控服务，用于监测服务器的运行状态、端口占用情况、内存使用、CPU负载等关键指标。

#### 1.1.1 启动监控服务

```bash
# 启动监控服务
start start_monitor.bat
```

#### 1.1.2 监控数据

监控数据保存在以下文件中：
- `logs/monitor.log`: 监控日志
- `logs/monitoring_data.json`: 监控数据

#### 1.1.3 监控指标

监控服务会监测以下指标：
- CPU使用率
- 内存使用率
- 磁盘使用率
- 网络流量
- 进程数量
- 后端服务状态
- 数据库文件状态
- 上传目录状态

### 1.2 异常告警

监控服务会在以下情况发出告警：
- CPU使用率超过80%
- 内存使用率超过80%
- 磁盘使用率超过80%
- 后端服务未运行
- 数据库文件不存在
- 上传目录不存在

## 2. 故障排查

### 2.1 常见故障

#### 2.1.1 后端服务启动失败

**可能原因**：
- 端口被占用
- 配置错误
- 依赖缺失

**解决方案**：
- 检查端口占用情况：`netstat -ano | findstr :8001`
- 停止占用端口的进程：`taskkill /PID <PID> /F`
- 检查配置文件是否正确
- 安装缺失的依赖：`pip install -r requirements.txt`

#### 2.1.2 文件上传失败

**可能原因**：
- 文件格式不支持
- 文件大小超过限制
- 上传目录权限不足
- 解析失败

**解决方案**：
- 检查文件格式是否为JSON/EVTC/ZEVTC
- 检查文件大小是否超过配置的限制
- 检查上传目录权限
- 查看日志文件了解具体错误

#### 2.1.3 评分计算失败

**可能原因**：
- 配置错误
- 数据格式无效
- 计算逻辑错误

**解决方案**：
- 检查配置文件是否正确
- 检查日志文件了解具体错误
- 验证输入数据格式

### 2.2 日志分析

#### 2.2.1 日志文件

- `logs/backend.log`: 后端服务日志
- `logs/monitor.log`: 监控服务日志
- `logs/error.log`: 错误日志

#### 2.2.2 日志级别

- DEBUG: 详细的调试信息
- INFO: 一般信息
- WARNING: 警告信息
- ERROR: 错误信息
- CRITICAL: 严重错误信息

## 3. 备份与恢复

### 3.1 数据库备份

#### 3.1.1 手动备份

```bash
# 复制数据库文件
copy database\gw2_logs.db database\gw2_logs_backup_$(date +%Y%m%d_%H%M%S).db
```

#### 3.1.2 自动备份

建议设置定时任务，定期备份数据库文件。

### 3.2 配置备份

#### 3.2.1 手动备份

```bash
# 复制配置文件
copy config\scoring_rules.json config\scoring_rules_backup_$(date +%Y%m%d_%H%M%S).json
```

#### 3.2.2 自动备份

系统会在修改评分规则时自动创建备份。

### 3.3 恢复操作

#### 3.3.1 数据库恢复

```bash
# 恢复数据库文件
copy database\gw2_logs_backup_<timestamp>.db database\gw2_logs.db
```

#### 3.3.2 配置恢复

```bash
# 恢复评分规则配置
copy config\scoring_rules_backup_<timestamp>.json config\scoring_rules.json
```
